Act as stage 2 of a local, source-bound daily-log pipeline.

Write the final answer in correct, natural Polish.

TASK
Generate only:
1. DAILY_LOG content.
2. README_PL content.

Do not create files yourself.
The Python/Bash pipeline creates files, MERGED_SOURCES, manifests, validation, and distribution.

SOURCE BOUNDARY
Use only RAPORT_POSTEPOW and optional rough_work supplied by the pipeline.

DAILY_LOG RULES
The DAILY_LOG must document only work actually supported by RAPORT_POSTEPOW.

Required structure:

# DAILY LOG — YYYY-MM-DD

## 1. Zakres dnia

## 2. Praca faktycznie wykonana

## 3. Artefakty utworzone lub zmodyfikowane

## 4. Decyzje operacyjne i metodologiczne

## 5. Niepewności i nierozstrzygnięte punkty

## 6. Następne kroki

RULES
- Do not invent actions, files, decisions, paths, commits, or completion.
- Do not describe another date as work performed today.
- Do not place planned work in "Praca faktycznie wykonana".
- In "Następne kroki", write plain bullets without "status: planned".
- Do not include routing/placement metadata unless the report explicitly supports it.
- Do not copy conversational instructions such as "Pobierz, podmień", "uruchom ponownie", or "Rozumiem i zgadzam się".
- Do not treat diagnostic validator messages as work performed.
- If the report is weak or contradictory, preserve uncertainty instead of forcing completeness.
- If an artifact is mentioned only by old/historical context, do not classify it as created or modified today.
- If a line says something was not checked, not confirmed, or not done, do not mark it as completed.
