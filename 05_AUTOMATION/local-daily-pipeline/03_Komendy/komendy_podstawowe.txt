# KOMENDY PODSTAWOWE — lokalny pipeline daily log

Ten plik jest krótką ściągą do ręcznej pracy z pipeline’em dziennym w WSL / Ubuntu.

Hermes został usunięty z procesu. Obecny pipeline działa lokalnie przez Ollamę.

Najważniejsze rozróżnienie:
- PowerShell używa ścieżek typu: `C:\Users\...`
- WSL / Ubuntu używa ścieżek typu: `/mnt/c/Users/...`

Nie wklejaj ścieżek `/mnt/c/...` do PowerShella.

## 1. Wejście do katalogu projektu w WSL / Ubuntu

```bash

pwd
ls -la
```

## 2. Wejście do katalogu projektu w PowerShell

```powershell
cd "C:\Users\aleks\Documents\prompt-systems-lab\05_AUTOMATION\local-daily-pipeline"

```

## 3. Utworzenie folderu dnia

Zmień datę w zmiennej `DAY`.

```bash
BASE="/mnt/c/Users/aleks/Documents/prompt-systems-lab/05_AUTOMATION/local-daily-pipeline"
DAY="2026-04-25"
mkdir -p "$BASE/11_Wejscia_Dzienne/$DAY"
```

## 4. Wymagane wejście dnia

W folderze:

```text
11_Wejscia_Dzienne/YYYY-MM-DD/
```

musi być plik:

```text
chat_export_YYYY-MM-DD.md
```

Opcjonalnie może być:

```text
rough_work_YYYY-MM-DD.md
```

Brudnopis musi mieć małe litery w nazwie: `rough_work`, nie `ROUGH_WORK`.

## 5. Szybka walidacja wejść

```bash
BASE="/mnt/c/Users/aleks/Documents/prompt-systems-lab/05_AUTOMATION/local-daily-pipeline"
DAY="2026-04-25"

DAYDIR="$BASE/11_Wejscia_Dzienne/$DAY"
CHAT="$DAYDIR/chat_export_$DAY.md"
ROUGH="$DAYDIR/rough_work_$DAY.md"
WF="$BASE/10_Zrodla_Sterujace/COMMIT_WORKFLOW_MAP.md"
SCHEMA="$BASE/10_Zrodla_Sterujace/schemat_daily_log.md"
P1="$BASE/09_Prompty/generator_raportu_postepow.txt"
P2="$BASE/09_Prompty/generator_dokumentacji_dnia_v2.txt"

[ -d "$DAYDIR" ] && echo "DAYDIR=TAK" || echo "DAYDIR=NIE"
[ -f "$CHAT" ] && echo "CHAT=TAK" || echo "CHAT=NIE"
[ -f "$ROUGH" ] && echo "ROUGH_WORK=TAK" || echo "ROUGH_WORK=NIE / OPCJONALNIE"
[ -f "$WF" ] && echo "WORKFLOW=TAK" || echo "WORKFLOW=NIE"
[ -f "$SCHEMA" ] && echo "SCHEMA=TAK" || echo "SCHEMA=NIE"
[ -f "$P1" ] && echo "PROMPT_ETAP_1=TAK" || echo "PROMPT_ETAP_1=NIE"
[ -f "$P2" ] && echo "PROMPT_ETAP_2=TAK" || echo "PROMPT_ETAP_2=NIE"
```

## 6. Uruchomienie pipeline’u

```bash
bash "/mnt/c/Users/aleks/Documents/prompt-systems-lab/05_AUTOMATION/local-daily-pipeline/05_AUTOMATION/local-daily-pipeline/run_daily_pipeline.sh" 2026-04-25 20000 2000
```

Znaczenie parametrów:
- `2026-04-25` — dzień do przetworzenia,
- `20000` — chunk size, czyli rozmiar jednego kawałka tekstu,
- `2000` — overlap, czyli zakładka między chunkami.

Nie ustawiaj overlap większego lub równego chunk size.

## 11. Najczęstsze błędy

- Brak `chat_export_YYYY-MM-DD.md` blokuje pipeline.
- Brak `rough_work_YYYY-MM-DD.md` nie blokuje pipeline.
- Zła nazwa brudnopisu, np. `ROUGH_WORK_...`, nie zostanie wykryta.
- Ścieżki ze spacjami zawsze dawaj w cudzysłowie.
- Nie uruchamiaj komend WSL w PowerShellu bez przeróbki ścieżek.
- Jeżeli `DAILY_LOG` trafia do `02_Do_Naprawy`, to nie jest finalnie zatwierdzony.
